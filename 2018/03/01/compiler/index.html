编译原理

发表于 2018-03-01 | 分类于 compiler

术语

  1. 编译程序
    将高级语言程序翻译成目标机上的可执行文件
  2. 解释程序
    对高级语言程序逐行翻译成目标机上的可执行代码同时运行每一条翻译结果
  3. 翻译程序
    将一种编程语言翻译成另一种编程语言

计算思维

编译原理涉及计算机科学诸多领域, 例如自动机理论, 正则理论. 同时它也包含一系列计算思维, 所谓计算思维是指以计算机的基本术语和概念描述和理解现实世界的一种逻辑思维, 这些计算思维包括:

  1. 抽象
  2. 自动化
  3. 分解
  4. 递归
  5. 权衡与折衷

抽象即去除不必要的部分找出事物的本质, 例如图灵机就是对计算过程的抽象, 形式文法, 有限自动机等都是对抽象的应用. 自动化即将抽象的结果应用于实际, 利用计算机将其实现, 例如语法分析就是自动化的应用. 分解即将大问题分解成能解决的小问题, 通过对小问题的解决得到大问题的解决方案. 编译器将编译过程分成多个不同的阶段就是对分解的应用. 递归也是对分解的应用, 但它分解之后的子问题与原问题具有相似的结构, 递归下降语法分析就是对递归的应用.

编译过程

词法分析 -> 语法分析 -> 中间代码生成 –> 编译器优化 –> 目标代码生成

词法分析将输入字符串分解成一个个有意义的单词, 并进行分类, 描述模型为: 有限自动机
语法分析将单词串组成一个个语法模块, 描述模型为: 上下文无关文法
中间代码生成根据语义规则进行初步的翻译, 描述模型为: 3元组或4元组
编译器优化根据程序的等价变换原则对中间代码进行优化, 为目标代码生成做准备
目标代码生成根据目标机器的体系结构和指令含义的不同进行不同形式的目标代码生成

程序语言的定义

  1. 语法
  2. 语义
  3. 语用

语法包含两个层次:

  1. 词法规则(单词符号的形成规则, 根据语言而不同)
  2. 语法规则(语法单元的形成规则, 根据语言而不同)

词法规则用有限自动机描述, 语法规则用上下文无关文法描述

i.e:
E -> i
E -> E + E
E -> E * E
E -> (E)
为表达式的语法规则

语义描述了程序的意义, 称为语义规则, 可以用自然语言描述和属性文法描述语义.

程序语言的分类

  1. 过程式
  2. 应用型
  3. 逻辑型
  4. 面向对象型

程序语言的特点

结构

  1. 是否允许嵌套
  2. 作用域

语言需要考虑是否支持嵌套, 作用域如何确定, 以及作用域与嵌套的关系.

程序语言的语法描述

程序语言区别于自然语言, 但是可以通过形式化的方法来描述, 即上下文无关文法.

上下文无关文法

基本概念

如果有一个字母表, 包含一组特定的字母集合, 称之为∑, 我们将字母表中的字母组合称为一个字, 且∑*称为字母表的闭包, 其中包含所有字母表中的字母组成的字的集合.

字集的连接运算

若U,V是∑*的子集, U, V的连接运算表示为:

UV = {ab|a∈U, b∈V}
U在自身上的n次连接记:
U^n = UUUUU…(n个)
特殊的:
U^0 = {ε}, 其中ε表示字长为零的空字.
v的闭包:
v* = v^0∪v^1∪v^2…
v的正规闭包:
v+ = vv* 唯一可能的不同在于正规闭包不包含空字.

上下文无关文法定义

上下文无关文法G由四个要素:

  1. 终结符集合
  2. 非终结符集合
  3. 文法的起点
  4. 产生式集合
    即:
    G = (Vt,Vn,S,P), 其中Vt表示终结符集合, Vn表示非终结符集合,S表示作为文法起点的非终结符 S∈Vn , P表示产生式规则的有限集合, 如:

G = ({i, +, *,(,)}, {E}, E, P), 其中P = {
E -> i,
E -> E+E,
E -> E*E,
E -> (E)
}
描述的是包含标识符,加号,乘号,左右括号的表达式的文法规则. 对于 P 中的产生式, 左边是被定义的非终结符, -> 表示定义, 右侧是终结符和非终结符的组合, 其表示左侧的定义. 若左侧的被定义符号相同, 则可以简写:

E -> i|E+E|E*E|(E)

直接推出

当有产生式: A -> v, 则A可以直接被定义v替换, 如: aAb => avb, 在前面产生式成立时表示 aAb直接推出avb, 其中a, b ∈ (Vt∪Vn)*

推导

对 a1 经过多次直接推出, a1 => a2 => …. => an, 则称这个序列是a1 到an的一个推导,也称a1可以推导出an.

特别的, 用 a1 =>+ an 表示a1到an的一次及以上的推导, a1 =>* an 表示a1到an的0次或若干次的推导.所以有: a1 =>* an 等价于 a1 = an 或 a1 =>+ an.

语言

句型: S => a, a为非终结符称a为一个句子, 从文法起点S出发能够得到的所有句子集合称为文法G的语言 L(G):

L = {a | S =>+ a, a ∈ Vt*}

例如:

语言 L(G) = {a^mb^n| 1<=n<=m<=2n } 其对应的上下文无关文法是:
S->ab|aab
S->aSb|aaSb

语法树与二义性

如果将从文法起点开始的每一次最左或最右推导以树形纪录, 则是一颗语法树, 其中最左推导和最右推导的区别是, 最左推导每次找左边的第一个非终结符根据产生式进行替换, 最右推导与此相反. 如对表达式文法生成的句子: (i*i+i)

其最左推导与最右推导的语法树均为:

不同的句型推导得到相同的语法树, 说明语法树是句型推导的共性抽象.

文法的二义性

如果一个文法对于一个句子有两个不同的最左推导或有两个不同的最右推导, 则这个文法是二义性文法.也就是说一个句子在一个文法下的语法树并不唯一.

语言的二义性

如果一个语言找不到一个非二义性文法来描述, 则这个语言是二义性的. 即:

L(G) = L(G’), 而G是二义性的, G’是非二义性的, 则语言是非二义性语言.

二义性问题是不可判定问题, 即不存在一个算法, 其在有限步骤内确定一个文法是否是二义性文法, 但是可以找到一系列无二义性文法的充分条件

程序设计语言对上下文无关文法的限制

  1. 不存在 P -> P 这样的产生式
  2. 每一个非终结符 P 都必须有意义, 即: A =>* aPD, P =>* b, 其中 a,b ∈ Vt*, P,D ∈ Vn

乔姆斯基形式语言体系

乔姆斯基是形式语言体系的开创者, 他将形式语言分为4类: 分别是0,1,2,3型文法, 它们对语言的描述能力越来越弱, 但条件越来越苛刻.

0型文法(用图灵机识别):

A -> b, A,b ∈ (Vt∪Vn)*

1型文法(上下文相关文法用线性界限自动机识别):

A -> b, A,b ∈ (Vt∪Vn)*, 且|A| < |b|, A -> ε除外.

2型文法(上下文无关文法用非确定下推自动机识别):

A->b, A ∈ Vn, b ∈ (Vt∪Vn)*

3型文法(正规文法用有限自动机识别):

A->bC或A->b, A,C ∈ Vn, b ∈ Vt*, 其中C始终是最左侧或最右侧的唯一非终结符.

描述能力比较例子

对于语言 L = {a^nb^n|n>=1},其不能由正规文法产生, 因为其文法: S -> aSb|ab 并不满足正规文法的定义只能满足上下文无关文法的定义, 所以 L 不能由正规文法产生却能由上下文无关文法产生, 同样的对于语言 L = {a^nb^nc^n|n>=1}, 其不能由上下文无关文法产生却能由上下文相关文法产生.

程序设计语言既不是上下文无关语言也不是上下文相关语言, 这是因为有些部分只能由0型文法描述.但是描述语言的语法时仍然使用上下文无关文法, 对于上下文无关文法不能描述的部分约束使用其他方法来解决, 这是计算思维”权衡”的体现.
