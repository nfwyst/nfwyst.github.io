angularJs

发表于 2017-05-10 | 分类于 前端

Angular 作为前端框架的一种, 它为前端提供了一套 MVC, 即控制器, 视图, 模块, 能够让我们解耦我们的代码, 也让项目更加容易维护, 下面一起来认识一下

模块

如果我们想要将 angular 应用于我们的应用中, 我们需要定义一个模块, 模块是指代码的集合, 很多模块合起来就是我们的整个项目, 模块用于划分我们的整个应用, 如果我们的应用有很多功能, 我们可以将这些功能进行拆分, 然后交给不同的模块去处理.

控制器

控制器用于为视图(这里的视图指 html 文档)提供数据, angular 中有两种方式:

  1. 使用 $scope
  2. 使用 this

控制器并不能直接为 html 视图提供数据, 因为控制器中的数据都是其本身的局部变量, 不能被外部访问, 而在视图中只能访问 $scope 和 控制器实例, 所以需要将 $scope 作为依赖注入到控制器中并将数据挂载到 $scope 或 this 上.

表单

angular 为表单提供了一套机制, 可以使用 name 指定 一个名称, angular 会自动生成一个 ngModelController, 每一个名称对应一个 ngMOdelController, 这些 Controller 都有以下属性:

  • $error
  • $touched
  • $submited

$error 包含表单出错的一些信息比如required, $touched 表示表单是否被更改过, $submited 表示表单是否被提交
